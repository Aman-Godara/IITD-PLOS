COL729 Lab 1 : Clang Optimisations

In this assignment, we will examine the effect of different compiler optimization passes on program performance.

Instructions

  • Build SPEC CPU 2017 benchmarks with Clang for 32-bit x86
  • Experiment with different optimization permutations and combinations and observe the difference made by each optimization option (or pair of optimization options), etc. We would like to evaluate the depth and detail in which you have performed this investigation
    • You can compile a C program to LLVM bitcode using the following command (similar commands should be available for other languages) 
      clang -target i386 -emit-llvm <compiler-options> foo.c
      Please check the clang documentation on your own.
    • You can link multiple bitcode files into a single bitcode file using the following command 
      llvm-link foo.bc bar.bc
    • You can run optimization passes on the LLVM bitcode using the following command 
      opt <optimization-options> foo.bc
      Please check the opt tool's documentation on your own.
    • Generate code from LLVM bitcode to the target assembly (32-bit or 64-bit x86): 
      llc foo.bc
Prepare a report summarizing your findings: Your report should be succinct yet insightful. It should talk about the most consequential optimizations (to runtime) and the least consequential optimizations. Do some optimizations depend on each other: which ones? How does the order of application of the optimisations impact the runtime?
